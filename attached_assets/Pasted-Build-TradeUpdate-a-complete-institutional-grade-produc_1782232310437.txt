PART 5 — INSTITUTIONAL SCORING ENGINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ScoringEngine class with 7 analysis layers.
Minimum 50 candles before any scoring.
Show "Gathering market data..." until ready.
Never show 0.0 scores — always show loading state.

Total score: 0-50 points

LAYER 1 — HIGHER TIMEFRAME TREND (15m)
Calculate: EMA9, EMA21, EMA50, ADX(14)

Trend Score (0-10):
10: EMA9>EMA21>EMA50 AND ADX>30 (strong bull)
   OR EMA9<EMA21<EMA50 AND ADX>30 (strong bear)
8: EMA9/EMA21 aligned AND ADX 25-30
6: EMA9/EMA21 aligned AND ADX 20-25
3: Mixed EMA alignment
0: ADX < 20 (choppy — avoid)

Store: trendDirection = BULL|BEAR|NEUTRAL

LAYER 2 — SMART MONEY CONCEPTS (15m+5m)
Detect:
- Market structure: HH/HL (bull) or LH/LL (bear)
- Break of Structure (BOS): close beyond last swing
- Change of Character (CHoCH): first opposing BOS
- Order Blocks: last opposite candle before BOS
- Fair Value Gaps: gap between candle 1 and 3

SMC Score: add up to +2 to Trend Score if:
- Structure aligns with EMA trend
- Price near order block in trend direction
- No CHoCH in last 10 candles

LAYER 3 — VOLATILITY EXPANSION (5m)
Calculate: BB(20, 2σ), Band Width, ATR(14)

Volatility Score (0-10):
10: Bands expanding >8% vs 10 bars ago
   AND price within 0.5σ of band edge
   AND ATR increasing
8: Bands expanding 4-8% AND near band edge
5: Bands neutral
2: Bands contracting (squeeze forming)
0: Hard squeeze — avoid entry

Store: bandTouched = UPPER|LOWER|NONE
Store: atrValue for stop calculation

LAYER 4 — ENTRY TIMING (5m+1m)
RSI(14) on 5m:
+3.5 if RSI > 72 (sell) or RSI < 28 (buy)
+2 if RSI > 65 or RSI < 35
+0 if RSI 35-65
RSI divergence: +1 bonus if detected

Stochastic(5,3,3) on 5m:
+3 if %K>82 (sell) or %K<18 (buy) AND crossed %D
+1.5 if %K>75 or %K<25

MACD(12,26,9) on 5m:
+3.5 if histogram crosses zero in trade direction
+2 if histogram trending but not crossed
MACD divergence: +0.5 bonus

Entry Timing total: 0-10
Minimum 2 of 3 indicators must score > 0

LAYER 5 — PULLBACK QUALITY (1m+5m)
Valid buy zone:
Price within 0.1% of EMA21 on 5m
OR price at middle BB
OR price at detected Order Block
OR price in Fair Value Gap

Pullback Score (0-10):
10: In pullback zone AND rejection wick on 1m
7: In pullback zone, no rejection yet
4: Slightly outside pullback zone
0: Not near pullback zone

LAYER 6 — CANDLE PATTERN (1m)
Check last 3 closed 1m candles:
- Bullish/Bearish engulfing
- Pin bar (wick 2× body size)
- Inside bar breakout
- Doji + directional resolution
No pattern = cap total score at 25
Pattern confirmed = allow full score

LAYER 7 — RISK ALIGNMENT (dynamic)
Auto stop = 1.5 × signal candle height
Min 50 pips, Max 200 pips
TP1 = 1.5× stop (50% close)
TP2 = 3× stop (remaining 50%)
Min R:R = 2:1

Risk Score (0-10):
10: No daily loss, zero consecutive losses,
   stop 50-200 pips, R:R ≥ 2:1, within limits
7: 1 consecutive loss, still within limits
5: Approaching daily loss limit (>70% used)
2: 1 more loss stops bot
0: Daily loss hit OR 2+ consecutive losses
   OR stop outside range OR R:R < 2:1

SIGNAL DIRECTION (all must be true):
BUY: trendDirection=BULL, bandTouched=LOWER,
   RSI<40, Stoch<30, Pullback≥4, 
   No CHoCH last 10 candles, Pattern confirmed
SELL: trendDirection=BEAR, bandTouched=UPPER,
   RSI>60, Stoch>70, Pullback≥4,
   No CHoCH last 10 candles, Pattern confirmed

ADAPTIVE INTELLIGENCE (after 50 trades):
Track accuracy per indicator per trade.
Recalculate weights every 25 trades after that.
If indicator accuracy < 45%: weight × 0.85
If indicator accuracy > 65%: weight × 1.10
Cap: max 1.5× original, min 0.5× original
Apply user's adaptive weights to scoring.

RANGE CONTEXT FILTER:
Calculate 30-day high and low.
Top 20%: BUY scores × 0.85, SELL scores × 1.10
Bottom 20%: SELL scores × 0.85, BUY scores × 1.10
Middle 60%: no adjustment

SPIKE FILTER (always active):
If any 1m candle range > 3× ATR(14):
Classify as spike candle
Cancel pending entries
Move open trade stop to break even immediately
Wait 2 full 1m candles then re-enable
Status: "⚡ Spike detected — waiting 2 candles"

CONSOLIDATION DETECTOR:
If last 8 candles on 5m have range < 1.5× ATR:
Disable trend-following entries
Enable breakout mode:
If price closes beyond consolidation by 0.5× ATR:
Enter immediately on breakout candle close
Suspend pullback entry rule for breakout trades
Status: "◼ Consolidation — awaiting breakout"

FIRST CANDLE RULE:
Wait for first full 15m candle after session open
If first candle range > 2.5× ATR: wait 2 candles
Status: "⏱ Waiting for first session candle — Xm"

FALSE BREAKOUT FILTER:
Price closes outside BB then back inside next candle
= false breakout — reject, log reason

COUNTER-TREND MODE (Reversal strategy only):
Requires: RSI>78 (sell) or RSI<22 (buy)
Requires: BB breach ≥ 2.5σ
Requires: Confirmed rejection wick on 1m
Stop: beyond extreme wick high/low
Target: middle BB only
Max risk: 0.75% regardless of profile

PART 10 — BACKTESTING ENGINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BacktestService:
Uses stored candle history from database
Default period: 30 days
Runs scoring engine on historical candles
Simulates trade execution with paper P&L

Process:
1. Load 30 days of R_75 candle data
2. Replay candle by candle chronologically
3. Run scoring engine on each candle
4. Simulate entries, stops, TPs, time stops
5. Track running balance, peak, drawdown
6. Log every simulated trade
7. Calculate all metrics

Results:
- Total trades, wins, losses
- Win rate percentage
- Profit factor (gross wins / gross losses)
- Maximum drawdown (% and $)
- Total simulated P&L
- Equity curve (JSON array of balance points)
- Trades by session (count + win rate each)
- Score range performance table:
  25-29 | 30-34 | 35-39 | 40-44 | 45-50
  (trades, win rate, avg P&L per range)
- Indicator accuracy in backtest
- Best trade, worst trade, avg duration
- Sharpe ratio (if > 1.5 = good strategy)

Side-by-side comparison view:
Left: Backtest results (new strategy)
Right: Live performance (current strategy)
Comparison metrics highlighted:
Green: Backtest better than live
Red: Backtest worse than live
Shows: "Backtest suggests +12% improvement
in win rate vs current live performance"

Export: PDF report with all charts and metrics

PART 11 — AUTH + SIGNUP SYSTEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

POST /api/auth/signup
- Validate username (3-20 chars, alphanumeric)
- Validate email format
- Validate password strength
- Validate country provided
- Hash password with bcrypt
- Save to pending_signups table
- Send SIGNUP_RECEIVED email to user
- Create admin notification in admin panel
- Save to /server/data/signups.csv
- Return: { success, message: "Application submitted" }

POST /api/auth/login
- Check pending_signups first:
  If found and status=pending: 
  Return 403 { status: 'pending' }
  If found and status=rejected:
  Return 403 { status: 'rejected' }
- Check users table
- If user found but status=pending:
  Return 403 { status: 'pending' }
- Validate password
- Check account lockout (5 fails = 15 min lock)
- Generate JWT (8 hour expiry, httpOnly cookie)
- Log successful login with IP
- Return: { role, username, mustChangePassword,
           hasProfile, hasToken }

POST /api/auth/logout
- Clear JWT cookie server-side
- Log logout event
- Return 200

POST /api/admin/signups/approve/:id
- Move from pending_signups to users table
- Set status=approved, is_active=1
- Assign safe profile
- Send APPROVED email to user
- Log admin action

POST /api/admin/signups/reject/:id
- Update pending_signups status=rejected
- Send REJECTED email to user
- Log admin action

Rate limiting:
Auth routes: 10 requests/minute/IP
All other routes: 60 requests/minute
